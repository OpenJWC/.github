# OpenJWC 项目详细介绍

## 1. 项目背景与需求分析

### 1.1 高校教务信息获取现状

在当前数字化校园建设进程中，高校教务信息的管理和发布虽然已经实现了信息化，但学生在获取教务信息时仍然面临诸多挑战：

- **信息分散**: 教务信息分散在官网、教务系统、学院通知等多个渠道
- **更新不及时**: 传统通知方式存在滞后性，学生难以及时获取最新信息
- **检索困难**: 缺乏有效的搜索功能，查找特定信息费时费力
- **移动性差**: 官网访问体验在移动设备上不佳，不便随时随地查看
- **信息过载**: 大量无关信息干扰，难以快速获取重要通知

### 1.2 市场现有解决方案分析

目前市场上存在一些教务信息获取解决方案，但都存在一定的局限性：

#### 1.2.1 学校官方教务系统

- **优点**: 权威可靠，信息完整
- **缺点**: 界面陈旧，用户体验差，功能单一

#### 1.2.2 数智东南APP

- **优点**: 功能丰富
- **缺点**: 界面臃肿，反应迟钝

### 1.3 OpenJWC的解决方案

OpenJWC针对上述问题，提供了一套完整的私有化部署解决方案：

- **统一信息聚合**: 自动爬取并整合多渠道教务信息
- **智能搜索推荐**: 基于AI的智能搜索和个性化推荐
- **多终端覆盖**: Android、QQ Bot等多平台支持
- **隐私安全**: 私有化部署，数据完全自主可控
- **开源免费**: 完全开源，无广告，免费使用

## 2. 核心功能详解

### 2.1 智能信息爬取系统

#### 2.1.1 技术实现

OpenJWC的爬虫系统采用Rust语言开发，具有以下技术特点：

- **高性能并发**: 使用rayon库实现并行爬取，大幅提升爬取效率
- **智能解析**: 使用scraper库进行HTML解析，准确提取关键信息
- **增量更新**: 基于哈希值的增量更新机制，避免重复爬取
- **错误重试**: tenacity库实现的智能重试机制，提高爬取稳定性

#### 2.1.2 爬取策略

- **定时任务**: 可配置的定时爬取策略，支持多种时间间隔
- **优先级队列**: 重要信息优先爬取，确保关键通知及时更新
- **内容过滤**: 基于关键词和规则的内容过滤，剔除无关信息
- **格式转换**: HTML到Markdown的智能转换，保留格式和链接

### 2.2 AI智能对话系统

#### 2.2.1 RAG技术架构

OpenJWC采用了先进的RAG（Retrieval-Augmented Generation）架构：

```
用户提问 → 向量检索 → 相关文档召回 → LLM生成 → 回复用户
```

#### 2.2.2 技术实现细节

- **向量数据库**: 使用ChromaDB存储和检索文档向量
- **嵌入模型**: 调研Embedding 3模型，优化检索准确度
- **上下文管理**: 智能的上下文窗口管理，支持长对话
- **知识库更新**: 实时更新知识库，确保信息的时效性

#### 2.2.3 应用场景

- **自然语言查询**: "下周一有哪些考试？"
- **条件筛选查询**: "计算机学院的课程安排有哪些？"
- **相关信息推荐**: "关于补考的通知有哪些？"

### 2.3 多终端用户界面

#### 2.3.1 Android客户端

- **Material Design 3**: 采用最新的Material Design设计规范
- **Jetpack Compose**: 现代化的声明式UI框架
- **流畅交互**: 流畅的用户界面和交互动画
- **离线支持**: Room数据库实现离线缓存聊天记录
- **个性化设置**: 主题切换、字体大小、通知设置等

#### 2.3.2 Web管理面板

- **响应式设计**: 适配桌面和移动设备
- **实时监控**: 系统运行状态的实时监控面板
- **数据可视化**: ECharts图表展示系统数据
- **权限管理**: 细粒度的用户权限控制

#### 2.3.3 QQ机器人

- **群组推送**: 自动向指定QQ群推送新通知
- **命令交互**: 支持命令查询特定信息
- **定时提醒**: 考试、选课等重要事件提醒

### 2.4 系统管理功能

#### 2.4.1 用户管理

- **设备注册**: 支持多设备注册和管理
- **API密钥**: 客户端API密钥的生成和管理
- **访问控制**: 基于角色的访问控制（RBAC）

#### 2.4.2 内容管理

- **内容审核**: 对爬取的内容进行审核和管理
- **分类管理**: 支持多级分类和标签管理
- **敏感信息**: 敏感信息的过滤和处理

#### 2.4.3 系统监控

- **性能监控**: CPU、内存、磁盘等系统资源监控
- **访问统计**: 用户访问量、API调用量等统计数据
- **日志管理**: 详细的系统日志记录和查询

## 3. 技术优势与创新点

### 3.1 技术优势

#### 3.1.1 微服务架构

- **模块化设计**: 各服务独立部署、独立扩展
- **技术异构**: 不同服务可采用最适合的技术栈
- **容错性**: 单个服务故障不影响整个系统
- **可维护性**: 代码结构清晰，便于维护和升级

#### 3.1.2 异步处理

- **高并发**: 支持高并发访问
- **高性能**: 充分利用系统资源，提升系统性能
- **可扩展性**: 水平扩展能力强，支持用户量增长

#### 3.1.3 数据持久化

- **双数据库**: SQLite关系数据库 + ChromaDB向量数据库
- **数据备份**: 自动化数据备份和恢复机制
- **数据一致性**: 事务处理确保数据一致性

#### 3.1.4 容器化部署

- **环境一致性**: 容器化确保开发、测试、生产环境一致
- **快速部署**: 一键部署，简化运维流程
- **资源隔离**: 容器级别的资源隔离和限制

### 3.2 创新点

#### 3.2.1 RAG架构应用

将先进的RAG架构应用到教务信息检索领域，实现了智能化的知识问答系统，这在同类产品中较为罕见。

#### 3.2.2 多技术栈整合

在一个项目中整合了Python、Rust、Kotlin、JavaScript等多种编程语言和框架，体现了技术多样性和综合运用能力。

#### 3.2.3 私有化部署

完全支持私有化部署，用户可以完全控制自己的数据，这在数据隐私日益重要的今天具有重大意义。

#### 3.2.4 开源生态

完全开源，便于社区贡献和二次开发，降低了技术门槛，促进了技术普及。
