## 快速部署（推荐）

### 1. 安装Docker和Docker Compose

#### Ubuntu/Debian系统

```bash
# 更新软件包索引
sudo apt-get update

# 安装Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 启动Docker服务
sudo systemctl start docker
sudo systemctl enable docker

# 将当前用户添加到docker组
sudo usermod -aG docker $USER

# 安装Docker Compose
sudo apt-get install docker-compose-plugin

# 验证安装
docker --version
docker compose version
```

#### CentOS/RHEL系统

```bash
# 安装Docker
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 启动Docker服务
sudo systemctl start docker
sudo systemctl enable docker

# 将当前用户添加到docker组
sudo usermod -aG docker $USER

# 验证安装
docker --version
docker compose version
```

### 2. 克隆部署仓库

```bash
# 克隆OpenJWC-Server仓库
git clone https://github.com/OpenJWC/OpenJWC-Server.git
cd OpenJWC-Server
```

### 3. 配置管理员账号

```bash
# 创建管理员配置文件
cat > admins.json << EOF
[
  {
    "username": "admin",
    "password": "Admin@12345"
  }
]
EOF
```

**重要说明**：

- `admins.json`文件用于初始化管理员账号
- 系统启动时会自动读取此文件并同步到数据库
- 管理员登录后可通过控制面板修改密码
- 如需多个管理员，可在文件中添加多个账号对象

### 4. 创建docker-compose.yml

在项目根目录创建`docker-compose.yml`文件：

```yaml
version: '3.8'

services:
  backend:
    image: openjwc-backend:v1.0
    container_name: openjwc_backend
    restart: always
    volumes:
      - ./data:/app/data
      - ./logs:/app/logs
      - ./admins.json:/app/admins.json:ro
    environment:
      - TZ=Asia/Shanghai

  crawler_worker:
    image: openjwc-crawler:v1.0
    container_name: crawler_worker
    restart: always
    volumes:
      - ./data:/app/data
      - ./logs:/app/logs
    environment:
      - TZ=Asia/Shanghai
    command: ["uv", "run", "python", "-m", "app.crawler_wrapper"]
    depends_on:
      - backend

  frontend:
    image: openjwc-frontend:v1.0
    container_name: openjwc_frontend
    restart: always
    ports:
      - "8000:8000"
    depends_on:
      - backend
```

### 5. 启动服务

```bash
# 拉取镜像
docker compose pull

# 启动所有服务
docker compose up -d

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f
```

### 6. 访问系统

- **Web管理面板**: `http://your-server-ip:8000`
- **默认管理员账户**:
  - 用户名: `admin`
  - 密码: `Admin@12345`

> ⚠ **安全提示**: 首次登录后请立即修改默认密码！

> ⚠ **浏览器安全警告**: 如果服务器没有SSL证书，需要在浏览器中将服务器IP添加为安全上下文。在Edge浏览器中访问 `edge://flags/#unsafely-treat-insecure-origin-as-secure`，启用对应字段并添加服务器IP。

## 手动部署

如果您需要更细粒度的控制或开发调试，可以选择手动部署各个组件。

### 1. 后端服务部署

#### 1.1 克隆后端仓库

```bash
git clone https://github.com/OpenJWC/OpenJWC-webapi.git
cd OpenJWC-webapi
```

#### 1.2 配置管理员账号

创建`admins.json`文件：

```bash
cat > admins.json << EOF
[
  {
    "username": "admin",
    "password": "Admin@12345"
  }
]
EOF
```

#### 1.3 编译爬虫二进制文件

```bash
# 安装Rust工具链
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env

# 克隆爬虫仓库
git clone https://github.com/OpenJWC/JwcCrawler.git
cd JwcCrawler

# 编译发布版本
cargo build --release

# 复制二进制文件到后端目录
mkdir -p ../OpenJWC-webapi/bin
cp target/release/jwc-crawler ../OpenJWC-webapi/bin/
```

#### 1.4 安装Python依赖

```bash
cd ../OpenJWC-webapi

# 安装uv包管理器
curl -LsSf https://astral.sh/uv/install.sh | sh

# 创建虚拟环境并安装依赖
uv sync
```

#### 1.5 启动后端服务

```bash
# 开发模式启动
uv run uvicorn main:app --reload --host 0.0.0.0 --port 8080

# 生产模式启动
uv run uvicorn main:app --host 0.0.0.0 --port 8080 --workers 4
```

#### 1.6 使用CLI工具管理

```bash
# 进入CLI管理界面
python main.py

# 常用命令：
create user <name> <max_devices>  # 创建API密钥
create admin <username> <password> # 创建管理员
show apikeys                       # 列出所有密钥
crawl                              # 运行爬虫任务
sync                               # 同步管理员配置
```

### 2. 前端服务部署

#### 2.1 克隆前端仓库

```bash
git clone https://github.com/OpenJWC/OpenJWC-web-frontend.git
cd OpenJWC-web-frontend
```

#### 2.2 安装依赖

```bash
npm install
```

#### 2.3 开发模式启动

```bash
npm run dev
```

#### 2.4 生产构建

```bash
# 构建生产版本
npm run build

# 使用nginx部署
# 将dist目录内容复制到nginx的web根目录
sudo cp -r dist/* /var/www/html/
```

### 3. 爬虫部署（手动运行）

#### 3.1 使用爬虫二进制文件

```bash
# 基本用法
./bin/jwc-crawler -o output.json -d 2024-01-01

# 参数说明：
# -o, --out: 输出文件路径（必需）
# -d, --date: 指定起始日期（可选），格式：YYYY-MM-DD，未指定则爬取所有数据
# --data-source: 数据源，可选值为 jwc、xsxy、cs（默认：jwc）
# --with-contents-only: 只爬取包含内容的通知
```

#### 3.2 数据源说明

- `jwc`: 教务处官网
- `xsxy`: 新生学院
- `cs`: 计算机学院

#### 3.3 使用系统包装器

```bash
# 运行完整的爬取任务（爬取->保存JSON->同步数据库->同步向量库）
python -m app.crawler_wrapper

# 或在CLI中运行
python main.py
# 然后输入：crawl
```

### 4. QQ机器人部署

#### 4.1 克隆机器人仓库

```bash
git clone https://github.com/OpenJWC/openjwc-qqbot.git
cd openjwc-qqbot
```

#### 4.2 安装依赖

```bash
# 使用uv安装依赖
uv sync
```

#### 4.3 配置机器人

```bash
# 复制配置模板
cp .env.example .env

# 编辑配置
nano .env
```

**配置示例**：

```bash
# OneBot配置
ONEBOT_ACCESS_TOKEN=your-access-token
ONEBOT_WS_URLS=["ws://127.0.0.1:3001"]

# OpenJWC后端配置
OPENJWC_AUTH_KEY=sk-your-api-key
OPENJWC_HOST=your-server-ip
OPENJWC_PORT=8000
OPENJWC_USE_HTTP=true
OPENJWC_DEVICE_ID=qq_bot_device_01

# QQ群配置（用于@全体成员通知）
OPENJWC_NOTICE_GROUPS=[123456789,987654321]

# 检查间隔（秒）
OPENJWC_CHECK_INTERVAL=5
```

#### 4.4 启动机器人

```bash
uv run bot.py
```

## 配置说明

### 管理员账号配置

管理员账号通过`admins.json`文件进行初始化：

```json
[
  {
    "username": "admin",
    "password": "Admin@12345"
  },
  {
    "username": "another_admin",
    "password": "Another@12345"
  }
]
```

**重要说明**：

- 这是增删管理员账号的唯一手段
- 系统启动时会自动同步此文件到数据库
- 只有文件中列出的管理员可以访问控制面板
- 管理员登录后可通过控制面板修改密码

### 系统设置配置

系统设置存储在数据库的`system_settings`表中，可通过以下方式配置：

#### 方式一：Web控制面板

1. 登录管理面板
2. 进入"系统设置"页面
3. 修改相应配置并保存

#### 方式二：CLI命令

```bash
uv run main.py

# 查看所有设置
echo

# 修改设置
set deepseek_api_key your-deepseek-api-key
set zhipu_api_key your-zhipuai-api-key
set crawler_interval_minutes 480
set crawler_days_gap 200
```

#### 默认配置项

| 设置键 | 默认值 | 说明 |
|--------|--------|------|
| `deepseek_api_key` | `this_is_your_dpsk_key` | DeepSeek API密钥 |
| `zhipu_api_key` | `this_is_your_glm_key` | 智谱AI API密钥 |
| `crawler_interval_minutes` | `480` | 爬虫执行间隔（分钟） |
| `crawler_days_gap` | `200` | 爬取日期范围（天） |
| `search_max_day_diff` | `60` | 搜索最大日期差（天） |
| `search_min_similarity` | `0.3` | 搜索最小相似度 |
| `prompt_debug` | `False` | 提示调试模式 |
| `notices_auth` | `False` | 通知访问需要认证 |
| `prompt_preview_length` | `500` | 提示预览长度 |
| `submission_max_length` | `10000` | 提交内容最大长度 |
| `system_prompt` | （见默认值） | 系统提示词 |

### 爬虫配置说明

爬虫配置通过系统设置进行：

```bash
# 爬虫间隔（分钟）
set crawler_interval_minutes 480  # 每8小时执行一次

# 爬取日期范围（天）
set crawler_days_gap 200  # 爬取最近200天的通知
```

### Docker配置详解

```yaml
version: '3.8'

services:
  # 后端服务
  backend:
    image: openjwc-backend:v1.0
    container_name: openjwc_backend
    restart: always
    volumes:
      - ./data:/app/data              # 数据库文件持久化
      - ./logs:/app/logs              # 日志文件持久化
      - ./admins.json:/app/admins.json:ro  # 管理员配置（只读）
    environment:
      - TZ=Asia/Shanghai              # 时区设置

  # 爬虫服务
  crawler_worker:
    image: openjwc-crawler:v1.0
    container_name: crawler_worker
    restart: always
    volumes:
      - ./data:/app/data              # 数据库文件共享
      - ./logs:/app/logs              # 日志文件共享
    environment:
      - TZ=Asia/Shanghai
    command: ["uv", "run", "python", "-m", "app.crawler_wrapper"]
    depends_on:
      - backend

  # 前端服务
  frontend:
    image: openjwc-frontend:v1.0
    container_name: openjwc_frontend
    restart: always
    ports:
      - "8000:8000"                    # 端口映射
    depends_on:
      - backend
```

## 常见问题

### Q1: Docker服务无法启动

**问题**: `docker compose up -d` 启动失败

**解决方案**:

```bash
# 检查Docker服务状态
sudo systemctl status docker

# 启动Docker服务
sudo systemctl start docker

# 查看详细错误日志
docker compose logs

# 检查端口占用
sudo netstat -tulpn | grep :8000
```

### Q2: 后端API无法访问

**问题**: 无法访问 `http://localhost:8000`

**解决方案**:

```bash
# 检查后端服务状态
docker compose ps backend

# 查看后端日志
docker compose logs backend

# 检查防火墙设置
sudo ufw allow 8000
# 或
sudo firewall-cmd --add-port=8000/tcp --permanent
```

### Q3: 管理员无法登录

**问题**: 使用`admins.json`中的账号无法登录

**解决方案**:

```bash
# 检查admins.json文件格式
cat admins.json

# 重启后端服务以同步管理员配置
docker compose restart backend

# 查看日志确认同步成功
docker compose logs backend | grep admin
```

### Q4: 数据库初始化失败

**问题**: 数据库初始化时出现错误

**解决方案**:

```bash
# 检查数据库目录权限
ls -la data/

# 修改权限
sudo chown -R $USER:$USER data/

# 数据库会自动初始化，无需手动操作
# 查看日志确认初始化状态
docker compose logs backend | grep "数据库"
```

### Q5: AI功能不可用

**问题**: AI聊天功能报错

**解决方案**:

```bash
# 检查API密钥配置
python main.py
echo

# 测试API连接（在CLI中）
set deepseek_api_key your-api-key

# 检查网络连接
diagnose
```

### Q6: 爬虫无法获取数据

**问题**: 爬虫运行但无法获取到数据

**解决方案**:

```bash
# 手动运行爬虫测试
python main.py
crawl

# 查看爬虫日志
docker compose logs crawler_worker

# 检查爬虫配置
echo
# 确认crawler_days_gap设置合理
```

## 故障排除

### 健康检查

```bash

docker compose ps

# 检查后端健康状态
curl http://localhost:8080/

# 检查前端健康状态
curl http://localhost:8000/

docker stats
```

### 日志查看

```bash
# 查看所有服务日志
docker compose logs

# 查看特定服务日志
docker compose logs backend
docker compose logs crawler_worker
docker compose logs frontend

# 实时查看日志
docker compose logs -f backend

# 查看最近100行日志
docker compose logs --tail=100 backend
```

### 性能优化

```bash
# 清理未使用的Docker资源
docker system prune -a

# 限制容器资源使用
# 在docker-compose.yml中添加:
services:
  backend:
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
        reservations:
          cpus: '1'
          memory: 1G
```

### 数据备份

```bash
# 备份数据库
docker compose exec backend cp data/jwc_notices.db /tmp/jwc_notices.db
docker cp openjwc_backend:/tmp/jwc_notices.db ./backup/jwc_notices_$(date +%Y%m%d).db

# 备份向量数据库
docker compose exec backend tar -czf /tmp/chroma_db.tar.gz data/chroma_db
docker cp openjwc_backend:/tmp/chroma_db.tar.gz ./backup/chroma_db_$(date +%Y%m%d).tar.gz

# 备份管理员配置
cp admins.json ./backup/admins_$(date +%Y%m%d).json
```

### 数据恢复

```bash
# 恢复数据库
docker cp ./backup/jwc_notices_20240115.db openjwc_backend:/tmp/jwc_notices.db
docker compose exec backend cp /tmp/jwc_notices.db data/jwc_notices.db

# 恢复向量数据库
docker cp ./backup/chroma_db_20240115.tar.gz openjwc_backend:/tmp/chroma_db.tar.gz
docker compose exec backend tar -xzf /tmp/chroma_db.tar.gz -C data/

# 恢复管理员配置
cp ./backup/admins_20240115.json admins.json
docker compose restart backend
```
