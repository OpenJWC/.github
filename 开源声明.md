## 贡献指南

### 如何贡献

我们欢迎任何形式的贡献，包括但不限于：

- Bug修复
- 新功能开发
- 文档改进
- UI/UX优化
- 性能优化
- 测试用例编写
- 国际化翻译

### 贡献流程

#### 1. Fork项目

在GitHub上Fork本项目到您的账户。

#### 2. 创建分支

```bash
git checkout -b feature/your-feature-name
```

#### 3. 进行开发

遵循项目的代码规范进行开发。

#### 4. 提交代码

```bash
git add .
git commit -m "feat: add new feature"
```

#### 5. 推送分支

```bash
git push origin feature/your-feature-name
```

#### 6. 创建Pull Request

在GitHub上创建Pull Request，并详细描述您的改动。

### 代码规范

#### Python代码规范

遵循PEP 8规范，使用Black进行代码格式化。

```bash
# 格式化代码
black app/

# 检查代码格式
black --check app/
```

#### JavaScript/TypeScript代码规范

使用ESLint和Prettier进行代码检查和格式化。

```bash
# 运行ESLint
npm run lint

# 格式化代码
npm run format
```

#### Rust代码规范

使用`cargo fmt`和`cargo clippy`。

```bash
# 格式化代码
cargo fmt

# 运行Clippy
cargo clippy
```

### 提交信息规范

遵循[Conventional Commits](https://www.conventionalcommits.org/)规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型(type):**

- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式（不影响代码运行）
- `refactor`: 重构（既不是新增功能，也不是修改bug）
- `test`: 增加测试
- `chore`: 构建过程或辅助工具的变动

**示例：**

```bash
feat(notices): add search functionality

- Implement full-text search
- Add search API endpoint
- Update UI with search bar

Closes #123
```

### Pull Request规范

创建Pull Request时，请提供：

1. **清晰的标题**: 使用上述提交信息规范
2. **详细的描述**: 说明改动的目的和内容
3. **相关Issue**: 引用相关的Issue编号
4. **测试说明**: 说明如何测试您的改动
5. **截图**: 如果涉及UI改动，提供截图

**Pull Request模板：**

```markdown
## 改动说明
简要说明这次改动的目的和内容。

## 改动类型
- [ ] Bug修复
- [ ] 新功能
- [ ] 重构
- [ ] 文档更新
- [ ] 其他

## 测试
- [ ] 单元测试已通过
- [ ] 集成测试已通过
- [ ] 手动测试已完成

## 相关Issue
Closes #123

## 截图
（如果涉及UI改动，请提供截图）
```
