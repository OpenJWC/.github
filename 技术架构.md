# OpenJWC 技术架构文档

## 1. 系统整体架构

### 1.1 架构设计理念

OpenJWC采用现代化的微服务架构，结合容器化部署和异步处理技术，构建了一个高性能、高可用、可扩展的教务资讯系统。

**核心设计原则：**

- **模块化**: 各组件职责单一，接口清晰
- **异步处理**: 充分利用异步IO提升性能
- **容错设计**: 单点故障不影响整体系统
- **可扩展性**: 支持水平扩展和垂直扩展
- **安全性**: 多层次的安全防护机制

#### 2.1.3 关键API设计

**移动客户端认证流程：**

```
1. 管理员在控制面板创建API Key → 设置拥有者和最大设备数
2. 管理员分发API Key → 用户获取API Key
3. 客户端使用API Key发送请求，同时携带设备ID → 通过X-Device-ID头
4. 服务端验证API Key有效性 → 检查设备绑定状态
5. 首次访问自动绑定设备 → 后续直接访问
6. 返回数据 → 增加请求计数
```

**AI对话API流程：**

```
1. 用户发送问题 → POST /api/v1/client/chat
2. 向量检索 → ChromaDB similarity_search
3. 上下文构建 → prompt_engine.build_context
4. LLM调用 → DeepSeek/智谱AI API
5. 响应返回 → 格式化输出
```

### 2.5 QQ机器人 (openjwc-qqbot)

#### 2.5.1 NoneBot2架构

**异步事件处理：**

```python
from nonebot import on_command
from nonebot.params import CommandArg
from .client import NewsFetcher
from .config import Config

# 命令处理器
search_cmd = on_command("search", aliases={"搜索通知"}, priority=5, block=True)

@search_cmd.handle()
async def handle_search(arg: Message = CommandArg()):
    query = arg.extract_plain_text().strip()
    if not query:
        await search_cmd.finish("请输入搜索关键词，例如：/search 奖学金")

    # 调用后端API
    fetcher = NewsFetcher(
        host=config.openjwc_host,
        port=config.openjwc_port,
        auth=config.openjwc_auth_key,
        device_id=config.openjwc_device_id,
        use_http=config.openjwc_use_http
    )

    results = await fetcher.search_notices(query=query, top_k=10)

    # 格式化回复
    if results:
        msg = f"🔍 关于"{query}"的智能搜索结果：\n"
        for i, res in enumerate(results, 1):
            msg += (
                f"\n{i}. 【{res.label}】{res.title}\n"
                f"   📅 {res.date} | 🎯 匹配度: {res.similarity_score:.2f}\n"
                f"   🔗 {res.detail_url}\n"
            )
        await search_cmd.finish(msg)
    else:
        await search_cmd.finish(f"🔍 未能找到与"{query}"相关的资讯。")
```

#### 3.2.2 向量检索策略

**向量检索策略：**

```python
def search_with_metadata(query: str, n_results: int = 10, min_similarity: float = None):
    # 1. 生成查询向量
    query_vector = self.get_embedding(f"{query}, 今日日期{date.today()}")

    # 2. 设置时间过滤条件
    cutoff_date = date.today() - timedelta(days=max_day_diff)
    where_clause = {
        "$and": [
            {"date_int": {"$gte": cutoff_date_int}},
            {"in_notices_table": {"$eq": True}},
        ]
    }

    # 3. 执行向量检索
    results = collection.query(
        query_embeddings=[query_vector],
        n_results=n_results,
        where=where_clause
    )

    # 4. 过滤低相似度结果并返回
    filtered_results = []
    for i in range(len(results['ids'][0])):
        similarity = 1 - results['distances'][0][i]
        if min_similarity is None or similarity >= min_similarity:
            filtered_results.append({
                'id': results['ids'][0][i],
                'similarity_score': similarity,
                'metadata': results['metadatas'][0][i]
            })

    return filtered_results
```

## 5. 安全架构

### 5.1 认证授权机制

#### 5.1.1 JWT认证流程

```
1. 用户登录 → 验证用户名密码
2. 生成Token → JWT signed with secret key
3. 客户端存储 → LocalStorage / SecureStorage
4. 请求携带 → Authorization: Bearer {token}
5. 服务端验证 → 验证签名和过期时间
6. 授权访问 → 根据权限访问资源
```

**JWT Payload结构：**

```json
{
  "sub": "admin_username",
  "exp": 1642320967
}
```

### 5.2 数据安全

#### 5.2.1 密码加密

**bcrypt加密：**

```python
import bcrypt

def hash_password(password: str) -> str:
    salt = bcrypt.gensalt()
    hashed = bcrypt.hashpw(password.encode('utf-8'), salt)
    return hashed.decode('utf-8')

def verify_password(password: str, hashed: str) -> bool:
    return bcrypt.checkpw(
        password.encode('utf-8'),
        hashed.encode('utf-8')
    )
```

## 9. 技术难点与解决方案

### 9.1 向量检索优化

**挑战：** 大规模向量检索效率低
**解决方案：**

- 使用ChromaDB的HNSW索引算法
- 实现分片存储和检索
- 增量更新向量索引
- 缓存热门查询结果

### 9.2 AI上下文管理

**挑战：** 多轮对话的上下文管理复杂
**解决方案：**

- 基于会话ID的上下文存储
- 滑动窗口机制管理历史对话
- 智能上下文压缩和总结
- 相关性评分过滤历史信息

### 9.3 爬虫反爬应对

**挑战：** 目标网站反爬机制
**解决方案：**

- 随机User-Agent和请求头
- 分布式代理IP池
- 请求频率限制和退避策略
- 验证码识别集成

### 9.4 多端数据同步

**挑战：** 多客户端数据一致性
**解决方案：**

- 基于WebSocket的实时推送
- 版本号机制实现增量同步
- 冲突检测和解决策略
- 离线数据缓存和合并
